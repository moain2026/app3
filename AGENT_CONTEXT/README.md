# AGENT_CONTEXT — نظام معرفة وسياق الوكلاء

> هذا المجلد هو "الذاكرة الحيّة" للمشروع. يُحدَّث مع كل جلسة عمل ليضمن أن أي
> وكيل لاحق يفهم كل شيء ويكمل دون فقدان سياق.

## 👉 ابدأ من الجذر
اقرأ أولاً **[`../AGENTS.md`](../AGENTS.md)** — ملف التوجيه الرئيسي (نقطة الدخول
الإلزامية لكل وكيل).

## الهيكل
| المجلد/الملف | المحتوى | متى تقرأه | متى تحدّثه |
|---|---|---|---|
| `knowledge/PROJECT_OVERVIEW.md` | نظرة شاملة وحالة المشروع | بداية أي جلسة | عند تغيّر الصورة الكبيرة |
| `sessions/SESSION_LOG.md` | سجل كل جلسة (أحدث = الأعلى) | **أول شيء** لمعرفة نقطة الاستئناف | **نهاية كل جلسة** |
| `decisions/DECISION_LOG.md` | كل قرار معماري ولماذا | قبل نقض/تغيير قرار | عند اتخاذ قرار مهم |
| `issues/ISSUES_LOG.md` | المشاكل وحلولها والوقاية | عند مواجهة مشكلة مشابهة | عند حلّ مشكلة |

## القاعدة الذهبية
> اكتب التوثيق كأن الوكيل التالي لن يكلّم المستخدم أبداً ولن يقرأ غير ملفاتك.
> يجب أن يفهم كل شيء من توثيقك وحده.

## علاقته بمجلد `analysis/`
- `analysis/` = **المرجع الفني الثابت** (ما استُخرج من الـ APK + الخطط). يتغيّر
  ببطء عند اكتشاف معلومة جديدة.
- `AGENT_CONTEXT/` = **الذاكرة الحيّة** (ماذا فعلنا، لماذا، ما المشاكل، أين نقف).
  يتغيّر مع كل جلسة.
